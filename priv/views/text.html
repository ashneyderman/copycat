Single-click on the person's name to start up a conversation or open a hidden one. You can have only two conversations visible at a time.

To the left of the person's name there is a button indicating status:

  • - indicates person is online
  • - indicates person is offline

If you want to produce multiple auto-generated messages on the interval of 2 seconds, use the circular arrow button next to close button. That is a toggle button, so clicking it once will start the message production, clicking it again will pause it.

KNOWN ISSUE 0: when a message arrives (from anyone on the roster) and conversation is not visible client does not open it up automatically. Messages arrive in the conversations that are not displayed just fine but user needs to open the conversation to see the arrived messages.

KNOWN ISSUE 1: do not use reload button of your browser.

KNOWN ISSUE 2: messages sent to users that are offline will be lost. We do not have offline message storage implemented on the backend.
